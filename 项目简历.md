## 核心亮点

### 1. 大型开源项目二次开发能力
- 在 **DeerFlow 数万行代码**的 monorepo 中新增 `influencer` 模块
- 严格遵循 `deerflow.*`（框架层）→ `app.*`（业务层）的**单向依赖边界**
- Agent 工具采用**工厂模式 + 闭包注入**，解耦 harness 层与 app 层
- 所有 ORM 变更通过 Alembic migration 管理

### 2. 多 Agent 并行协作架构
- 设计 **1 个 Lead Agent + 4 个专业 SubAgent** 的并行分析体系
- 粉丝画像分析师 / 内容风格分析师 / 商业表现分析师 / 风险合规扫描师
- 注册到 DeerFlow SubAgent Registry，利用其原生并行调度能力（MAX_CONCURRENT=3）
- 编写代码级降级方案（FallbackResult），实现"宁可少维度，不空白页"的容错策略

### 3. 四维加权评分 + 反馈自优化闭环
- 设计**匹配度 × 传播力 × 带货力 × 性价比**四维评分模型（权重 0.35/0.25/0.25/0.15）
- 品牌方提交合作反馈（1-5 星 + 评价 + 标签）后，自动对比预测分 vs 实际评分
- 偏差 > 阈值时通过**梯度下降简化算法**反向微调权重
- confidence 随反馈量单调递增，权重逐渐收敛

### 4. 全栈工程能力
- **后端 20+ 个 RESTful API**，含搜索、CRUD、分析、反馈、统计
- **前端 5 条路由 + 14 个组件**，SVG 雷达图、对比抽屉、报告下载
- **150 条种子数据**（MockDataAdapter），支持一键切换真实平台 API
- Service 层自定义异常体系 + Router 层统一错误处理 + 降级响应

## 面试话术建议

**Q: 介绍一下这个项目？**
> 我在字节开源的 DeerFlow AI Agent 框架上做了二次开发，构建了一个直播带货智能选达人平台。品牌方可以通过自然语言搜索达人，系统会自动调用评分引擎做四维打分，然后派遣 4 个专业 SubAgent 做并行深度分析，最后生成推荐报告。合作完成后有反馈闭环，评分模型会根据实际效果自动优化权重。

**Q: 你怎么处理 DeerFlow 的 harness/app 边界问题？**
> DeerFlow 规定 `deerflow.*` 不能 import `app.*`。我的 4 个 Agent Tool 定义在 harness 层的 `deerflow/tools/influencer/` 下，采用了工厂模式——工具函数接收 app 层的 adapter 和 engine 作为闭包参数，在 Gateway 启动时注入。这样工具代码不依赖任何 `app.*` 导入，通过了 CI 的 harness boundary 检查。

**Q: SubAgent 并行分析怎么做的？**
> 我定义了 4 个专业 SubAgent 类型（粉丝画像/内容风格/商业表现/风险合规），注册到 DeerFlow 的 BUILTIN_SUBAGENTS。Lead Agent 在分析达人时，通过系统提示词中的五阶段工作流指引，自动派遣最多 3 个 SubAgent 并行执行，任一完成后立刻启动风险扫描。代码级 fallback 保证了部分维度失败时仍能出结果。

**Q: 评分自优化怎么实现的？**
> 品牌方提交合作反馈后，FeedbackService 对比预测分和实际评分（rating×20 转为百分制）。偏差超过 10 分时按学习率 0.05 微调权重并归一化。confidence 随反馈次数递增，权重逐渐收敛。阈值设在 confidence≥0.6 才触发调整，避免早期数据噪声。
