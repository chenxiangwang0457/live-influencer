# 直播带货智能选达人平台 — 功能清单

> 基于 DeerFlow 二次开发 | Phase 1-4 完成
> 后端 116 tests | 前端 pnpm check 通过

---

## 启动方式

```bash
# 终端1: 后端
cd backend
DEER_FLOW_AUTH_DISABLED=1 PYTHONPATH=. PYTHONIOENCODING=utf-8 PYTHONUTF8=1 uv run uvicorn app.gateway.app:app --host 0.0.0.0 --port 8001 --reload

# 终端2: 前端
cd frontend
pnpm dev
```

浏览器打开 **http://localhost:3000**，左侧栏：

```
💬 对话
🤖 智能体
⏰ 定时任务
✅ 达人广场        ← Phase 1
📋 选人任务        ← Phase 2
📊 反馈分析        ← Phase 4
```

## 一、达人广场（搜索 + 筛选）

| 操作 | 步骤 | 预期 |
|------|------|------|
| 打开达人广场 | 点击左侧栏"达人广场" | 看到 150 位达人卡片（分页，每页12条） |
| 按品类筛选 | 品类下拉选"美妆" | 卡片实时过滤为美妆达人 |
| 按粉丝范围筛选 | 粉丝(万) 填 10-50 | 只显示 10万-50万 粉丝的达人（输入为万单位） |
| 按互动率筛选 | 互动率(%) 填 2-10 | 进一步过滤 |
| 按报价筛选 | 报价(元) 填最小值/最大值 | 按报价区间过滤 |
| 按关键词搜索 | 搜索框输入达人名称 | 名称或品类匹配 |
| 切换排序 | 排序下拉选"场均GMV" | 按 GMV 降序排列 |
| 分页浏览 | 点击"上一页"/"下一页" | 翻页浏览更多达人 |
| 进入详情 | 点击达人卡片 | 跳转到达人详情页 |
| 前往选人任务 | 点击右上角"选人任务"按钮 | 跳转到选人任务列表 |
| 重置筛选 | 点击"重置"按钮 | 清空所有筛选条件 |

## 二、达人详情页

| 操作 | 步骤 | 预期 |
|------|------|------|
| 查看基本资料 | 页面上方 | 头像、昵称、平台标识、品类标签、数据来源 |
| 查看能力评分 | SVG 四维雷达图 | 匹配度 / 传播力 / 带货力 / 性价比 |
| 查看关键数据 | 数据卡片区 | 粉丝数、互动率、场均GMV、场均销量、报价区间 |
| 查看粉丝画像 | 粉丝画像区 | 年龄分布（18-24/25-34/35+）、性别比例、Top城市 |
| 查看内容风格 | 内容风格区 | 风格标签徽章（如"教程""种草""测评"） |
| 查看合作历史 | 品牌合作历史区 | 时间线（品牌名称 + 合作年份） |
| 返回上一页 | 点击左上角"返回上一页" | 浏览器后退到之前页面 |

## 三、选人任务（创建 + 管理）

| 操作 | 步骤 | 预期 |
|------|------|------|
| 打开任务列表 | 点击左侧栏"选人任务" | 任务列表，含状态筛选和分页 |
| 按状态筛选 | 状态下拉选"进行中" | 只显示对应状态的任务 |
| 新建任务 | 点击"新建任务" → 填写名称+目标 → 创建 | 任务出现在列表 |
| 进入任务详情 | 点击任务行 | 跳转到任务详情页 |
| 编辑任务 | 点"编辑" → 修改标题/目标/状态 → 保存 | 信息实时更新 |
| 手动添加达人 | 点"手动添加达人" → 搜索 → 添加 | 达人出现在候选人表格 |
| 候选人表格 | 查看表格 | 昵称(可点击跳详情)/品类/粉丝/互动率/GMV/匹配度/状态 |
| 候选人排序 | 点击"匹配度"列头 | 按分数升降序切换 |
| 修改候选人状态 | 状态下拉 → 待联系/已联系/已选定/已淘汰 | 状态立即更新 |
| 移除候选人 | 点击垃圾桶图标 | 候选人从列表移除 |
| 勾选对比 | 勾选 ≥2 位达人 → 点"对比" | 侧滑抽屉：多人SVG雷达图叠加 + 6项指标对比表 |

## 五、反馈闭环

| 操作 | 步骤 | 预期 |
|------|------|------|
| 触发反馈表单 | 候选人状态改为"已选定" | 该达人下方出现反馈表单卡片 |

| 选择评价标签 | 点击标签（专业度高/配合度好/转化率高等） | 标签选中/取消切换 |
| 填写文字评价 | 文本框输入 | 自由填写合作体验 |
| 提交反馈 | 点"提交反馈" | 写入数据库 + 自动更新 influencer_scores + 权重微调 |

## 七、对话中的 Agent 协作（可选高级功能）

| 操作 | 步骤 | 预期 |
|------|------|------|
| 自然语言搜索 | 对话页输入"帮我找美妆类达人" | Agent 调用 search_influencers，聊天中渲染达人卡片 |
| 达人对比 | 对话页输入"对比这几位达人" | Agent 调用 compare_influencers，聊天中渲染对比表 |
| 生成报告 | 对话页输入"生成推荐报告" | Agent 调用 recommend_report，聊天中渲染报告卡片 |
| 并行分析 | Agent 自动判断 | 对 Top 达人派遣 fan-analyst/content-analyst/commercial-analyst 并行分析 |
| 风险扫描 | 分析完成后自动触发 | risk-scanner 检查粉丝质量/品牌安全 |

> 注：Agent 协作依赖 DeerFlow 的 subagent 系统和 LLM 配置。确保 `config.yaml` 中 subagent_enabled=true。

## 建议体验路径

```
达人广场
  → 品类选"美妆"，粉丝填 10-50
  → 点几个达人卡片看详情
  → 回达人广场，点右上角"选人任务"
创建选人任务
  → 新建任务"618美妆达人选拔"，目标"找3位中腰部美妆达人"
  → 手动添加 3 位达人
  → 点"AI推荐" → 查看报告 → 下载
  → 勾选 2 位达人 → 点"对比"
  → 改状态为"已选定"
提交反馈
  → 给已选定的达人打 4 星，选标签，填评价 → 提交
查看看板
  → 打开反馈分析 → 看评分分布
```
